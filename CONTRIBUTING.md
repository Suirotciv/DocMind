# Contributing to DocMind

Contributions are welcome.

## How to Contribute

### Report an issue

Open an issue with:
- a clear description of the problem
- steps to reproduce it
- your browser and Node version

### Submit a fix

1. Fork the repository
2. Create a branch: `git checkout -b fix/your-issue`
3. Make your changes
4. Run `npm run check` and confirm it passes
5. Submit a pull request with a short summary

### Improve documentation

Clarifying README steps, improving code comments, and adding usage examples are all useful.

## Guidelines

- Keep changes small and focused
- Use clear commit messages
- Explain the purpose of each change in the PR description
- Keep code style consistent with the existing codebase
